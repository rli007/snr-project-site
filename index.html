Abstract

As Advanced Placement (AP) exams approach each year, high school students often struggle to identify their weak areas and create effective, personalized study strategies. Existing resources such as prep books, online videos, and generic AI tools offer standardized support but fail to adapt to individual learning gaps. To address this, my project, LLAMA-P, builds a personalized, AP-specific study assistant specifically tailored for AP U.S. History preparation, with concepts generalizable to other AP subjects.

The system integrates Groq-hosted LLaMA 3 models with a custom-built Retrieval-Augmented Generation (RAG) pipeline that extracts relevant content from the official College Board Course and Exam Description (CED) PDF. This document is preprocessed using a chunking system, which segments and cleans historical content into overlapping passages stored in structured JSON format. This enables the model to ground its answers in authentic source material when generating multiple-choice questions, giving hints, or explaining concepts.

A lightweight persistent memory system tracks the user’s past responses, learning patterns, and common errors using a simple memory.txt backend. The LLM summarizes learnings and ideas from any recently finished chat. This memory is injected into the prompt to personalize feedback and reinforce difficult concepts over time. The tool also supports an interactive practice mode, complete with hint generation, instant feedback, and error-specific memory updates.

Unlike traditional tools, LLAMA-P offers an adaptive, self-improving study experience that is context-aware, historically accurate, and tailored for the AP exam. By combining retrieval-based context injection, prompt engineering, and student-specific memory, my assistant empowers AP students to study more efficiently and confidently, representing a concrete step toward scalable, AI-driven personalized education.

How It Works

Personalized Memory Tracking

The assistant uses a lightweight memory system to track each student's learning progress. By storing past question attempts and feedback in a local memory.txt file, the system identifies individual weaknesses and injects relevant learning patterns into future interactions. This enables personalized, targeted feedback without relying on cloud-based storage or exposing user data.

Curriculum-Aligned Data Extraction

To ensure historical accuracy and AP specificity, the system parses the official College Board APUSH Course and Exam Description (CED) PDF. A custom Python pipeline extracts text, corrects formatting issues, and splits content into overlapping chunks based on structure. Each chunk is tagged with rich metadata like period, title, and section type. This process guarantees that all generated content aligns directly with the AP curriculum.

Structured, Local Content Database

The assistant stores all extracted APUSH content as structured JSON files, grouped by historical period and section. This local database enables keyword-based retrieval through a custom Retrieval-Augmented Generation (RAG) system. When students ask questions or request practice problems, the system fetches the most relevant syllabus-aligned chunks to ground the LLM’s responses in trusted source material.
